# IT Help Center — SharePoint

A company-wide IT Help Center built on SharePoint and used by all employees — a
single hub for IT knowledge-base guides, FAQs, an IT request form, and
announcements.

> The live site is internal (behind company login), so this repo is the public
> write-up. Screenshots are sanitised — company names, logos, real names, and
> internal data are removed or blurred.

## The problem

Employees had no central place to find IT guides, and the same questions kept
reaching the IT team through scattered channels — chats, calls, walk-ups, and
email. There was no single source of truth for how-to guides, FAQs, or requests,
which meant repetitive support work and inconsistent answers.

## What I built

A structured Help Center on SharePoint with four areas:

- **Knowledge base** — IT how-to guides, organised by category so staff can find
  answers themselves.
- **FAQs** — concise answers to the most common questions, to deflect repeat
  tickets.
- **IT request form** — one consistent place to raise IT requests.
- **Announcements** — IT notices and updates in a single, visible spot.

I designed the site structure and information architecture, created and organised
the content, and set up the request workflow — and I maintain it.

## Outcome

- A single self-service hub for **all employees**.
- **Fewer repeat questions** to the IT team — common answers now live in the
  knowledge base and FAQs.
- One consistent channel for IT requests instead of scattered ones.

_Metrics to add once available: number of guides/articles, departments using it,
and any measured drop in repeat queries._

## Lessons learned

- Clear information architecture is what drives adoption — people only self-serve
  if they can find the answer fast.
- A small set of strong FAQs deflects the highest-volume tickets.
