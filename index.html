Company-Wide Remote Access Deployment

Led end-to-end deployment of Splashtop across the organisation. Configured access policies, enrolled devices, and trained staff — cutting remote support response time and eliminating the need for on-site visits for routine troubleshooting.

Outcome: Enrolled nearly 70 endpoints in Splashtop remote access, enabling consistent remote support across all departments.

Case study — sanitised

Context A renewable-energy company with an HQ and multiple remote sites, where routine IT support still meant travelling to the affected site.

Problem No standardised remote-support tooling — troubleshooting meant travel and delays across departments.

My role Owned the rollout end-to-end — tool selection, access-policy design, endpoint enrolment, and staff enablement.

Approach Defined role-based access policies, enrolled endpoints in waves to avoid disruption, enforced MFA and session logging, and ran short per-department training sessions.

Outcome Around 70 endpoints enrolled in Splashtop remote access — routine issues are now resolved remotely instead of requiring on-site visits.

Lessons Phased enrolment kept support available throughout the rollout, and mapping access policies to roles up front prevented over-permissioning later.

Proxmox Virtualisation & Backup Management

Managed Proxmox VE nodes — provisioning VMs and LXC containers for Active Directory and business workloads. Integrated Proxmox Backup Server with automated backups, snapshots, and replication for disaster recovery. Configured storage pools, resource monitoring, and firewall rules to secure and isolate production environments.

Outcome: Maintained reliable virtualisation across multiple nodes with scheduled PBS backups, snapshots, and replication for disaster-recovery readiness.

Case study — sanitised

Context Production virtualisation for Active Directory and business workloads across multiple Proxmox nodes.

Problem Workloads needed reliable provisioning, isolation, and a tested recovery path — not just backups that are assumed to work.

My role Provisioned and maintained the VMs and LXC containers, designed the backup and replication schedule, and owned disaster-recovery readiness.

Approach Configured storage pools and resource monitoring, isolated production with firewall rules, scheduled PBS backups and snapshots, and replicated critical workloads between nodes.

Outcome Reliable virtualisation across nodes, with scheduled PBS backups, snapshots, and replication in place for disaster-recovery readiness.

Lessons An untested backup is not a backup — restore checks matter as much as the schedule, and replication frequency is a direct trade-off against how much data you can afford to lose.

Network Infrastructure & SCADA Security

Monitored and maintained network infrastructure across HQ and multiple sites, overseeing switches, access points, IP phones, and firewall devices to ensure consistent uptime and connectivity. Supported SCADA/OT vulnerability assessments and infrastructure risk analysis, helping identify and prioritise security gaps.

Outcome: Sustained stable connectivity across HQ and remote sites while contributing to a stronger network and OT security posture.

Case study — sanitised

Context A multi-site network — HQ plus remote sites — with operational-technology (SCADA) segments running alongside corporate IT.

Problem Mixed IT/OT environments need consistent connectivity and clear segmentation, so a corporate-side issue can never reach control systems.

My role Maintained switches, access points, IP phones, and firewalls across sites, and supported the vulnerability-assessment and risk-analysis work on the OT side.

Approach Monitored uptime across sites, maintained and reviewed firewall rules, and assisted structured assessments of the network and OT segments.

Outcome Stable multi-site connectivity, with assessment findings prioritised into a concrete list of gaps and mitigations.

Lessons Segmentation only works when it is enforced everywhere — least privilege between IT and OT segments is what keeps corporate incidents away from control systems.
